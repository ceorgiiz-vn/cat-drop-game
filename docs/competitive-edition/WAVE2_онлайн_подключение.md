# Волна 2 — как включить настоящий онлайн (лидерборд + лиги)

Клиентская часть уже готова в `competitive-edition`:
- недельный результат считается локально (`getWeekKey`, `catdrop_week_<неделя>`);
- лиги Bronze→Diamond и панель лиги в окне «Awards & League» работают;
- есть готовые «крючки» для сервера — их нужно только подключить.

Сейчас лидерборд локальный (демо-имена), лиги считаются на устройстве. Чтобы игроки соревновались по-настоящему между собой, нужен онлайн. Два пути — можно любой.

---

## Путь A. Google Play Games Services (без своего сервера)

Подходит, если игра идёт **только под Android/Google Play**. Каркас в проекте уже есть (`js/play_games.js`).

Шаги (в Google Play Console):
1. Открыть свою игру → раздел **Play Games Services → Setup and management → Configuration**.
2. Создать **Leaderboards**: например «All-time» и «Weekly» (у GPGS-лидербордов есть встроенный сброс: daily / weekly / all-time — недельный делается штатно, свой код не нужен).
3. Скопировать **ID лидерборда** (вид `CgkI...`).
4. В **Credentials** настроить OAuth (Play Console проведёт по шагам) и скачать **`google-services.json`** → положить в `android/app/`.
5. В игре вписать реальный ID: в `js/game.js` заменить пустую строку
   `const GPGS_LEADERBOARD_ID = "";` на ваш `CgkI...`.
6. Пересобрать. Кнопка «Leaders» покажет нативную таблицу, счёт будет отправляться.

Плюсы: не нужен сервер, вход через Google. Минусы: только Google Play; **лиг GPGS не умеет** — дивизионы всё равно считаются на клиенте (как сейчас) или через путь B.

---

## Путь B. Свой лёгкий бэкенд (веб + Android, полный контроль, нужен для настоящих лиг)

Подходит, если хотите единый лидерборд и на сайте, и в приложении, и полноценные лиги с рангом внутри дивизиона.

Минимальный вариант без «настоящего сервера» — serverless (бесплатные тарифы): **Cloudflare Workers + KV**, **Supabase**, **Firebase Firestore** или **Vercel + KV**. Нужны два эндпоинта:
- `POST /score` — тело `{ name, score, week }` → сохранить/обновить лучший результат игрока за неделю.
- `GET /leaderboard?week=YYYY-Www` → вернуть топ-N `[{name, score}]`.

Куда подключать в коде (уже подготовлено):
- Отправка: функция `submitScore()` в `js/game.js` — туда добавить `fetch(API + "/score", { method:"POST", body: JSON.stringify({ name: GameState.player_name, score: GameState.score, week: getWeekKey() }) })`.
- Показ: `populateLeaderboardOverlay()` уже умеет грузить лидерборд через `fetch` (сейчас по `google_sheets_url`) — можно указать туда адрес эндпоинта или добавить константу `LEADERBOARD_API`.
- Неделя: ключ недели уже есть — `getWeekKey()` (ISO-неделя, вид `2026-W28`).
- Лиги: пороги в массиве `LEAGUES` (`game.js`). Сервер может считать ранг игрока внутри его лиги и возвращать «ты N-й в Gold» — клиент уже знает лигу по недельному результату.

Про честность очков: клиентские очки в браузере можно подделать. Для казуалки это обычно терпимо; если важно — добавить на сервере простую валидацию (лимит частоты, потолок правдоподобия, подпись сессии). Это отдельная небольшая задача.

---

## Рекомендация
- Если релиз только в Google Play и лиги устраивают клиентские — **Путь A** (быстро, без сервера): завести GPGS-лидерборды, вписать ID, положить `google-services.json`.
- Если нужен веб + честные онлайн-лиги с рангом — **Путь B**: поднять serverless-эндпоинт и подключить к готовым крючкам.

Когда выберете путь и заведёте сервис/бэкенд — дайте адрес эндпоинта или ID лидерборда, и я допишу подключение в копии (несколько строк в `submitScore`/`populateLeaderboardOverlay`) и проверю.
