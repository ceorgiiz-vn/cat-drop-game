# Каталог мониторинга Cat Drop — что ловим через «чёрный ящик» + Sentry

Рабочий документ. Составлен по результатам сканирования кода игры.
Цель: заранее прописать ВСЕ типы багов/отклонений, которые мы хотим ловить
автоматически на любых устройствах (свои 2 + тысячи в открытом тестировании).

Легенда способа ловли:
- **[авто]** — Sentry ловит сам, доработок почти не нужно.
- **[сторож]** — нужно написать watchdog: код, который распознаёт «почерк» глюка.
- **[метрика]** — просто цифра, которую пишем постоянно для контекста.
- **[фича]** — это не мониторинг, а отдельная доработка (помечаю, чтобы не путать).

Для КАЖДОГО срабатывания в отчёт кладём общий конверт:
тип события, время, паспорт устройства (модель, Android, версия WebView, плотность
пикселей, частота экрана, размеры экрана и viewport), версия игры, и последние
N секунд из «чёрного ящика» (кадры + что делала игра) — плюс данные, специфичные
для конкретного события.

## Категория 0. Технические падения (код ломается)

- **0.1 [авто]** JS-исключения — любой `throw`, прилетает с местом в коде.
- **0.2 [авто]** Необработанные ошибки промисов (`unhandledrejection`).
- **0.3 [сторож]** Ошибки загрузки ресурсов: спрайт/звук/шрифт не загрузился (`img.onerror`).
- **0.4 [сторож]** Ошибка регистрации/работы Service Worker.
- **0.5 [сторож]** Ошибки нативных плагинов Capacitor (Play Games, App).

## Категория 1. Производительность и плавность

- **1.1 [метрика]** FPS: текущий, средний, минимальный за окно.
- **1.2 [сторож]** Долгий кадр — детектор рывков/зависаний (максимум времени кадра
  за окно превысил порог). Это и есть ловля «зависа при падении» на Xiaomi.
- **1.3 [метрика]** Разбивка времени кадра: физика / отрисовка / прочее — кто виноват.
- **1.4 [сторож]** Спайк «догона»: за один кадр сделано слишком много шагов физики
  (после заминки движок навёрстывает — сам становится рывком).
- **1.5 [сторож]** Долгие задачи и паузы сборки мусора (long tasks).
- **1.6 [метрика]** Память (если устройство отдаёт): рост по ходу партии = утечка.
- **1.7 [метрика]** Число активных шариков в момент просадки.
- **1.8 [сторож]** Тепловой дрейф: устойчивое падение FPS по мере сессии (троттлинг).

## Категория 2. Физические аномалии (игра не падает, но ведёт себя неправильно)

- **2.1 [сторож]** Кот крутится на месте: большая угловая скорость + позиция почти
  не меняется несколько секунд. (Твой рыжий кот в углу.)
- **2.2 [сторож]** Сверхупругий разлёт: скорость шарика выше разумного предела, или
  резкий скачок скорости при столкновении, или суммарная «энергия» в стакане зашкалила.
- **2.3 [сторож]** Застревание/дрожь у стенки: прижат к границе, скорость есть, а сдвига нет.
- **2.4 [сторож]** Кот вне стакана: позиция за границами (туннелирование сквозь стенку).
- **2.5 [сторож]** «Взрыв физики»: NaN/бесконечность в позиции или скорости.
- **2.6 [сторож]** Вечное неслияние: два кота одного уровня долго соприкасаются, но не
  сливаются (застрявшая логика merge / `mergingBodyIds`).
- **2.7 [сторож]** Двойное слияние / гонка флага `isMerged`.
- **2.8 [сторож]** Утечка тел: число шариков растёт ненормально (тела не удаляются после merge).
- **2.9 [сторож]** «Нельзя дропнуть»: `canDrop` завис в false, текущий кот не сбрасывается.
- **2.10 [сторож]** Визуальный масштаб/wobble вышел за допустимый диапазон.

## Категория 3. Раскладка и соотношение сторон

Игра рисует стакан в фиксированных координатах 720×1280, а HUD (счёт, меню, бустеры) —
это HTML-элементы поверх. На нестандартных экранах (Fold в развёрнутом виде,
планшеты, сплит-скрин, вырезы) они могут наезжать.

- **3.1 [сторож]** HUD наезжает на стакан: пересечение областей HUD-элементов с зоной стакана.
- **3.2 [сторож]** Пункты меню перекрывают друг друга (раскрывающееся меню).
- **3.3 [сторож]** Модалка не влезает в экран / контент обрезан.
- **3.4 [метрика]** Экстремальное соотношение сторон: логируем viewport, вычисленные
  размеры игрового поля и коэффициент масштаба (особенно Fold: сложен/разложен, сплит-скрин).
- **3.5 [сторож]** Вырез/notch перекрывает элементы HUD.
- **3.6 [сторож]** Кнопки бустеров ушли за край / в недосягаемую зону.
- **3.7 [метрика]** Ориентация: ландшафт на телефоне (показ подсказки «поверни»).
- **3.8 [метрика]** Плотность пикселей: канвас замылен/растянут при экстремальном DPR.

## Категория 4. Состояние и сохранения

- **4.1 [фича]** Потеря имени/рекордов при переустановке. **Сейчас облачного сейва НЕТ.**
  Решение — Google Play Games Saved Games (Snapshots): привязать прогресс к аккаунту,
  чтобы после переустановки он подтягивался. Это главный риск потери прогресса.
- **4.2 [авто/сторож]** Порча save-данных: JSON не разобрался (уже есть try/catch — логировать).
- **4.3 [сторож]** Рассинхрон рекордов: очки растут, а highscore/today/year не обновились.
- **4.4 [сторож]** Битые данные сохранённой сессии (resume).
- **4.5 [сторож]** localStorage переполнен / запись не прошла (quota exceeded).
- **4.6 [метрика]** Persistent storage не выдан браузером (риск очистки под нехватку места).
- **4.7 [сторож]** Тихий вход в Play Games упал / сервис не сконфигурирован.
- **4.8 [сторож]** Экономика: fish_coins ушли в минус, или покупка без списания/списание без покупки.

## Категория 5. Ввод и взаимодействие

- **5.1 [сторож]** Тап не привёл к дропу (клик был, кот не сброшен).
- **5.2 [сторож]** Двойной дроп по одному тапу.
- **5.3 [сторож]** Аппаратная кнопка «назад» (Android) ведёт себя не так (диалог/выход).
- **5.4 [сторож]** Модалка не закрывается.
- **5.5 [сторож]** Бустер применился без списания монет / завис в режиме «выбери кота».

## Категория 6. Ресурсы и загрузка

- **6.1 [сторож]** Спрайт кота не загрузился надолго — рисуется цифра-заглушка вместо картинки.
- **6.2 [сторож]** Звук не декодировался / заметная задержка Web Audio.
- **6.3 [сторож]** Service Worker отдаёт устаревший кэш (`CACHE_NAME` не подняли) —
  рассинхрон версий файлов.
- **6.4 [сторож]** Шрифт Nunito не загрузился (текст съезжает/меняет размеры).

## Плюс: ручная кнопка / жест «тут что-то не так»

Для непредвиденного, чего мы не заложили в сторожей. Игрок (или ты при тесте) видит
странность — жест/кнопка отправляет последние N секунд из «чёрного ящика». В открытом
тестировании становится кнопкой «пожаловаться».

## Порядок работ (черновой)

1. Базовый «чёрный ящик» (кат. 1) + подключение Sentry + паспорт устройства (кат. 3.4).
2. Авто-падения (кат. 0) — почти бесплатно с Sentry.
3. Сторожа физики по твоим примерам (2.1, 2.2, 2.3), затем остальные из кат. 2.
4. Сторожа раскладки (кат. 3) — наезды HUD/стакана.
5. Сторожа состояния/ввода/ресурсов (кат. 4–6).
6. Ручная кнопка «что-то не так».
7. Отдельно как фича: облачный сейв (4.1).

Пороги всех сторожей настраиваем на 2 своих телефонах, чтобы в Sentry не летел мусор
из ложных срабатываний, — и только потом выкатываем на массу.
